### 并发任务调度器 (Concurrent Priority Task Scheduler)

> 难度与分类
>
> 困难 (Hard) | 异步编程 / 队列设计 / 性能优化

#### 题目描述
在现代前端应用中，为了防止资源耗尽（如浏览器 TCP 连接限制），我们需要控制异步任务的并发数。请实现一个 Scheduler 类，要求：

1. 并发控制：同时执行的任务数量不能超过 limit。
2. 优先级队列：任务带有优先级（数字越大优先级越高），高优先级任务应优先进入执行队列。
3. 超时机制：支持为单个任务设置超时时间，若超时则自动标记为失败并释放位置。
4. 状态追踪：能够查询当前正在执行的任务数和等待中的任务数。

#### 输入输出

```javascript
// constructor(limit: number): 初始化并发限制。
// add(task: () => Promise<any>, priority: number, timeout: number): 添加异步任务。返回一个 Promise，解析为任务结果或超时错误。
// 示例
const s = new Scheduler(2);

const addTask = (time, order, priority) => {
  s.add(() => new Promise(res => setTimeout(res, time)), priority)
    .then(() => console.log(`任务 ${order} 完成`));
};

// 限制为 2，同时加入 3 个任务
addTask(1000, '1', 1); // 立即执行
addTask(500, '2', 10); // 立即执行 (优先级最高)
addTask(300, '3', 5);  // 队列等待

// 输出预期：
// 500ms 后: 任务 2 完成，任务 3 开始
// 800ms 后: 任务 3 完成
// 1000ms 后: 任务 1 完成
```

**提示/约束**

*任务执行失败（Reject）不应阻塞调度器，应继续执行后续任务*  
*timeout 为可选参数，默认为无限*  
*必须考虑极端情况：如任务在等待队列中时就被取消或超时*  
*性能：避免在每次添加任务时对整个队列进行 O(n log n) 排序*

**解题思路**

`底层结构：`使用 最大堆 (Max Heap) 维护等待队列可以实现 O(log n) 的插入复杂度，优于数组排序。

`递归调度：`在任务完成的 finally 阶段，触发 next() 逻辑，从堆中取出最高优先级任务。

`超时竞速：`利用 Promise.race() 将业务逻辑 Promise 与一个 setTimeout 的计时 Promise 进行竞争，确保超时后能及时释放并发位。

`并发计数：`使用一个变量 runningCount 实时记录当前执行中的 Promise 数量。
