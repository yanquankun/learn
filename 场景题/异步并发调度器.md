### 题目：带优先级的异步并发调度器 (Priority Async Scheduler)

#### 题目描述
请实现一个 Scheduler 类，用于调度异步任务，满足以下要求：

1. 并发限制：构造函数接收 maxConcurrency（最大并发数），保证同时运行的任务不超过该限制。

2. 优先级调度：提供 add(taskFactory, priority) 方法。
 + taskFactory：返回 Promise 的函数。
 + priority：整数，数值越大优先级越高（默认为 0）。

3. 抢占机制：当并发池已满时，新任务进入等待队列。当有任务完成释放资源时，必须优先执行等待队列中优先级最高的任务（若优先级相同，则按照先来后到的顺序）。

#### 输入输出

```javascript
Input：
new Scheduler(2)
scheduler.add(() => fetch(...), 10)

Output：
add 方法返回一个 Promise，该 Promise 需在任务实际执行完成后 resolve 或 reject。
```

**示例 Case**

```javascript
const scheduler = new Scheduler(2); // 最多同时运行 2 个任务

// 辅助函数：创建一个耗时 time 毫秒的任务
const task = (time, name) => () =>
  new Promise(r => setTimeout(() => {
    console.log(`Finish ${name}`);
    r();
  }, time));

// 开始调度
scheduler.add(task(1000, '1. Low'), 1);    // T0: 立即运行 (池: 1/2)
scheduler.add(task(500,  '2. High'), 10);  // T0: 立即运行 (池: 2/2)
scheduler.add(task(300,  '3. Normal'), 5); // T0: 池满，入队 (等待中优先级: 5)
scheduler.add(task(400,  '4. High+'), 10); // T0: 池满，入队 (等待中优先级: 10)

// 预期输出顺序：
// 500ms -> Finish 2. High (释放一个槽位，检查队列)
//       -> 队列中有 [3(p=5), 4(p=10)]，4 优先级更高，即使 3 先来
//       -> 立即运行 4
// 900ms -> Finish 4. High+
//       -> 运行 3
// 1000ms -> Finish 1. Low
// 1200ms -> Finish 3. Normal
```

**提示与约束**

1. 任务一旦开始执行，不可中断或取消。
2. 只有在等待队列中的任务才会根据优先级重排序。
3. 需处理任务本身的 reject 情况，不能中断后续调度。

**解题思路**

`队列设计：`维护一个 waitingQueue。简单实现可用数组，每次取出前进行 sort；高性能实现应使用 最大堆 (Max Heap)，使入队出队复杂度降为 O(log N)。

`执行控制器：`add 方法返回一个新的 Promise，将该 Promise 的 resolve/reject 句柄保存在队列项中，以便在调度执行时触发。

`递归调度：`每当一个任务 finally（无论成功失败），都要触发 runNext()。
核心逻辑：runNext 检查 runningCount < maxConcurrency 且 waitingQueue 不为空，取出最高优先级项执行。

**代码实现**

```javascript
class Scheduler {
  maxConcurrency = null;
  runningCount = 0;
  // 由于需要保证同优先级任务的执行顺序，所以引入一个 order 属性来记录任务的添加顺序
  order = 0;
  waitingQueue = [];

  constructor(maxConcurrency) {
    this.maxConcurrency = maxConcurrency;
    this.runningCount = 0;
    this.waitingQueue = [];
  }

  add(task, priority = 0) {
    return new Promise((resolve, reject) => {
      const job = { task, resolve, reject, priority, order: this.order++ };
      if (this.runningCount < this.maxConcurrency) {
        this.run(job);
      } else {
        this.waitingQueue.push(job);
        // 优先级高的任务优先执行，同优先级的任务按照添加顺序执行
        this.waitingQueue.sort(
          (a, b) => b.priority - a.priority || a.order - b.order,
        );
      }
    });
  }

  run({ task, resolve, reject }) {
    this.runningCount++;

    // 防止传入的 task 不是一个Promise，统一用 Promise.resolve 包裹
    Promise.resolve()
      .then(task)
      .then(resolve, reject)
      .finally(() => {
        this.runningCount--;
        const nextJob = this.waitingQueue.shift();
        if (nextJob) this.run(nextJob);
      });
  }
}

const scheduler = new Scheduler(2); // 最多同时运行 2 个任务
const startTime = +new Date().getTime();

// 辅助函数：创建一个耗时 time 毫秒的任务
const task = (time, name) => () =>
  new Promise((r) =>
    setTimeout(() => {
      console.log(`${+new Date().getTime() - startTime}ms: Finish: ${name}`);
      r(name);
    }, time),
  );

// 开始调度
scheduler.add(task(1000, "1. Low"), 1).then(
  (name) => console.log("Task name:", name),
  (error) => console.log("Task error:", error),
); // T0: 立即运行 (池: 1/2)
scheduler.add(task(500, "2. High"), 10).then(
  (name) => console.log("Task name:", name),
  (error) => console.log("Task error:", error),
); // T0: 立即运行 (池: 2/2)
scheduler.add(task(300, "3. Normal"), 5).then(
  (name) => console.log("Task name:", name),
  (error) => console.log("Task error:", error),
); // T0: 池满，入队 (等待中优先级: 5)
scheduler.add(task(400, "4. High+"), 10).then(
  (name) => console.log("Task name:", name),
  (error) => console.log("Task error:", error),
); // T0: 池满，入队 (等待中优先级: 10)

// Output：
// 503ms: Finish: 2. High
// Task name: 2. High
// 912ms: Finish: 4. High+
// Task name: 4. High+
// 1002ms: Finish: 1. Low
// Task name: 1. Low
// 1213ms: Finish: 3. Normal
// Task name: 3. Normal
```
